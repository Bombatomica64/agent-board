# Product

## Register

product

## Users

Developers running several local AI coding agents across one or more repositories. They need to see active work, avoid duplicate effort, and resolve ownership conflicts without interrupting agent sessions.

## Product Purpose

Agent Board is a machine-local coordination surface for declaring, claiming, tracking, and auditing units of agent work. Success means a developer or agent can understand current ownership and recent activity at a glance, then act with minimal ceremony.

## Brand Personality

Utilitarian, compact, trustworthy. The interface should feel like a dependable developer tool whose state is easy to scan and whose actions are unambiguous.

## Anti-references

Avoid decorative dashboards, oversized metrics, ornamental gradients, novelty interactions, glass effects, and spacious marketing-style layouts. Do not make routine task coordination feel ceremonial.

## Design Principles

- Make ownership and lifecycle state unmistakable.
- Optimize for scanning several repositories and agents quickly.
- Keep common actions close to the state they affect.
- Surface failures explicitly instead of silently swallowing them.
- Prefer familiar controls and dense, stable layouts.

## Accessibility & Inclusion

Maintain practical keyboard focus, semantic labels, readable contrast, and reduced-motion support. Formal compliance certification is not currently required.
